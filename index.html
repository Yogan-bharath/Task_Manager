Browser Rendering Pipeline

Understanding how browsers transform raw HTML and CSS bytes into pixels on your screen — the complete step-by-step journey from network bytes to visual output.

• Parsing Phase
• Tree Building
• Rendering Phase
• Output Phase

01

🗒️
PARSING PHASE

HTML Bytes

Raw bytes from the server are decoded into characters using the specified encoding (UTF-8).

02

🔍
PARSING PHASE

Tokenization

Characters converted into distinct tokens: start/end tags, attributes, and content nodes.

03

🧩
PARSING PHASE

Node Creation

Each token is converted into a node object with properties, event listeners, and relationships.

04

🌳
TREE BUILDING

DOM Tree

Nodes linked into a hierarchical tree structure representing the entire page document.

05

🎨
TREE BUILDING

CSS Parsing

CSS bytes tokenized, rules parsed with selectors and declarations extracted.

06

🌿
TREE BUILDING

CSSOM Tree

CSS rules assembled into a CSS Object Model — a tree of style rules with cascading applied.

07

⚙️
Rendering Phase

Render Tree

DOM + CSSOM merged. Only visible nodes included. Each node carries computed styles.

08

📐
Rendering Phase

Layout

Browser calculates exact position, size, and geometry of every render tree element.

09

🖌️
Rendering Phase

Paint

Fill in pixels for text, colors, images, borders, and shadows on individual layers.

10

🔗
Output Phase

Compositing

Painted layers sent to GPU, composited in correct order using the stacking context.

11

💻
Output Phase

Screen Output

Final rendered bitmap displayed on the user's screen at ~60fps for smooth interaction.
